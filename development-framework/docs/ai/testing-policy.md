## When To Add Tests

Add or update tests when:

- Behavior changes
- A bug is fixed
- Permissions or auth behavior changes
- Data scoping changes
- API response behavior changes
- Business calculations change
- Background jobs or integrations change

## Minimum Validation

Before finishing AI-assisted work:

- [ ] Relevant tests added or updated, when possible
- [ ] Relevant tests executed
- [ ] Lint/typecheck executed, if available and relevant
- [ ] Manual validation steps documented, if needed
- [ ] Untested risks clearly reported

## If Tests Cannot Be Run

State clearly:

- Which tests were not run
- Why they were not run
- What should be run before merge
- Any risk caused by missing validation
